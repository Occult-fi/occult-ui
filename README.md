# occult-ui

Front-end for the [Occult](https://github.com/Occult-fi/occult) confidential batch-auction AMM on Solana.

Pure client-side React app. No backend, no server-side rendering — ZK proof generation runs in-browser via [`occult-wasm`](https://github.com/Occult-fi/occult-wasm), wallet signing via standard Solana wallet adapters.

## Stack

- Vite 7 + React 18 + TypeScript
- `@solana/web3.js`, `@solana/wallet-adapter-*`
- `react-router-dom` for routing (`/`, `/demo`, `/pools`, `/soon`)
- `comlink` to offload WASM proof generation to a WebWorker (~500 ms per Token-2022 confidential transfer)
- `occult-wasm` for the Phase B proof set (validity + equality + batched range proofs)
- Generated TS client (`src/protocol/generated/`) — Solita output from the shank-emitted IDL of the on-chain program

## Run

```bash
pnpm install
pnpm dev          # http://localhost:5173
pnpm build        # static dist/
```

Set the RPC endpoint via env:
```bash
VITE_RPC_URL=http://127.0.0.1:8899 pnpm dev
# or devnet:
VITE_RPC_URL=https://api.devnet.solana.com pnpm build
```

The UI reads `public/state.json` for program ID, primary pool, wrapper addresses, and the demo wallet's confidential token accounts. Generated by `occult/scripts/local-init.sh` on the localnet side and gitignored — never commit secrets.

## Pages

- **`/`** — Landing. Redacted hero, problem comparison, three-step explainer, animated swap demo, tech grid.
- **`/demo`** — Live swap. Wallet-driven `submit-order` Phase B flow (8 transactions: 3 ZK preflight + main with sibling Confidential Transfer + 3 cleanup). Batch ticker, anonymity-set counter, live tickets, claim flow — all backed by WS subscriptions on the pool, batch, and OrderTicket PDAs.
- **`/pools`** — LP management. Real on-chain pool discovery via `getProgramAccounts`; MyPositions decodes the user's confidential LP-token balance (AES-decrypt of `decryptable_available_balance`); Deposit / Withdraw modals run the 12-tx `AddLiquidity` / `RemoveLiquidity` orchestrators end-to-end.
- **`/soon`** — placeholder.

## Wallets

- **Phantom** (production path) — standard `wallet-adapter` integration, signs every tx the user submits.
- **DemoWallet** (`src/wallet/DemoWalletAdapter.ts`) — headless auto-signing wallet for the demo flow. Reads alice's keypair from `/demo-wallets/alice.json` so the multi-tx Phase B sequence runs without 11 popups. Disabled in any non-demo build.

## Architecture notes

- WS-backed live state (`useProgramAccounts`, `useAccountData`, `useSlot`) — no polling for batches, tickets, balances.
- Pool struct decode (`src/protocol/poolAccount.ts`) maps the shank-emitted account into a typed surface; LP-supply / reserves / volume / genesis-slot drive the APY and TVL panels.
- Proof generation lives in the worker (`src/wasm/worker.ts`); the main thread keeps AES decrypt for balance display only.
- Native SOL → shSOL wrapper has a 1:1 confidential mint; the `Wrap` / `Unwrap` flow rides the same Phase B proof pipeline.

## Design system

- Pure black ground (`#000`) + hairline whites
- Redaction-bar accent (`#1d1d1f`) — segmented, hover-to-reveal
- SF Pro Display / SF Mono via system stack
- 1440px page width, 160px section padding (compact: 96px)
- Single-color, no gradients, no warm accent

See `src/landing.css`, `src/demo.css`, `src/pools.css` for the full token system.
